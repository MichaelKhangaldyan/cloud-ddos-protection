# v2 — Բարելավված տարբերակ (ռեցենզիայից հետո)

Այս թղթապանակում գտնվում են բարելավված ծրագրային ֆայլերը, որոնք
պատասխանում են ռեցենզենտի մեկնաբանությանը արգելափակումների ավտոմատ
հանման անհրաժեշտության մասին:

## Ֆայլեր

| Ֆայլ | Նկարագրություն |
|------|----------------|
| `smart_block.py` | Բարելավված ավտոմատ արգելափակում (iptables + nginx, ավտոմատ unban) |
| `dashboard.py` | Թարմացված Dashboard countdown-ով և ban status-ով |
| `analytics_helper.py` | Հելպեր մոդուլ — `format_time_remaining()`, `load_bans()` |

## Բարելավումները

### 1. Կրկնակի պաշտպանություն (L3/L4 + L7)

```
Հին տարբերակ:           Նոր տարբերակ:

[Փաթեթ] → nginx → 403   [Փաթեթ] → iptables DROP (kernel-ի մակարդակ)
                                  → (եթե անցավ) → nginx → 403
```

iptables-ի DROP-ի շնորհիվ փաթեթները դեն են նետվում մինչև nginx-ին
հասնելը, ինչը ապահովում է բարձր կատարողականություն DDoS-ի ընթացքում:

### 2. Աստիճանական բան

```
Առաջին ստրայք   → 24 ժամ
Երկրորդ ստրայք → 5 օր
Երրորդ ստրայք  → 30 օր
```

Եթե IP-ն 30 օր ընթացքում նոր խախտում չի արել, ստրայք-ները վերակայվում են:

### 3. Ավտոմատ unban

Յուրաքանչյուր րոպե systemd timer-ով smart_block.py-ն ստուգում է
`bans.json`-ը: Երբ որևէ IP-ի `ban_until`-ը գերազանցել է ընթացիկ ժամանակը,
կանոնը հանվում է **և iptables-ից, և nginx-ից**:

### 4. Action logging

Բոլոր գործողությունները լոգավորվում են `/var/log/smart_block.log`
ֆայլում: Հնարավորություն է տալիս ետադարձաբար վերլուծել, թե երբ և
ինչու որևէ IP է բանի ենթարկվել:

## Աշխատանքի ընթացքը

```
┌──────────────────────────────────────────────────────────────┐
│ systemd timer (յուրաքանչյուր րոպե)                           │
│                                                              │
│  └→ smart_block.py                                           │
│        │                                                      │
│        ├─ 1. Կարդալ /var/log/nginx/attack.log                │
│        ├─ 2. Հաշվել 429-ները ըստ IP-ի (վերջին 200 գիծ)      │
│        ├─ 3. Թարմացնել bans.json-ը (ստրայք + ban_until)     │
│        ├─ 4. Կիրառել iptables DROP կանոնները                │
│        ├─ 5. Թարմացնել /etc/nginx/blocked_ips.conf-ը        │
│        ├─ 6. Reload Nginx (եթե փոփոխվել է)                  │
│        ├─ 7. Հանել պրոստրաթ բանարեր (auto-unban)            │
│        └─ 8. Մաքրել 30 օր հին գրառումները                   │
└──────────────────────────────────────────────────────────────┘
```
